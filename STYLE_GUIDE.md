# 執筆スタイルガイド

本書のすべての章は、このガイドに従って執筆する。教材の一貫性はこのファイルが保証する。

## 1. 基本原則

1. **教科書として読める完成度** — 箇条書きの羅列やメモで終わらせない。日本語の解説文で学習者を導く。
2. **1ファイル1テーマ** — 長くなりすぎる場合は `part-1` / `part-2` に分割する。
3. **日本語話者の視点** — 説明は常に「日本語と比べてどうか」「日本語話者がどこでつまずくか」を軸にする。
4. **正直さ** — 不確かな情報は書かない。事実情報(歴史・制度・統計)は [SOURCE_POLICY.md](./SOURCE_POLICY.md) に従い出典を付ける。
5. **CEFR 整合** — 各章の冒頭に対象 CEFR レベルを明示する。そのレベルで未習の文法・語彙を説明なしに使わない。

## 2. ルーマニア語の表記規則

- **正書法は DOOM3(Academia Română, 2021)に準拠**する。
- ダイアクリティクスは必ず正しい Unicode を使う:
  - `ș` (U+0219 コンマ付き) / `ț` (U+021B コンマ付き) — セディーユ付き `ş` `ţ` は**使用禁止**
  - `ă` (U+0103), `â` (U+00E2), `î` (U+00EE)
- **î / â の使い分け**(1993年正書法): 語頭・語末は `î`(în, a coborî)、語中は `â`(român, cânta)。複合語・接頭辞付きでは元の語頭の `î` を保つ(neînceput)。
- 動詞は辞書形を `a + 不定詞`(a fi, a avea)で示す。
- 名詞は「単数形 / 複数形」「性」を初出時に示す。

## 3. IPA・発音表記

- IPA は音素表記 `/ /` を基本とする。
- 主要対応: ă=/ə/, â・î=/ɨ/, ș=/ʃ/, ț=/t͡s/, j=/ʒ/, ce・ci=/t͡ʃ/, ge・gi=/d͡ʒ/, che・chi=/k/, ghe・ghi=/ɡ/, 語末の無音化しない -i=/ʲ/(前の子音の口蓋化)。
- アクセント位置は IPA 内で `ˈ` で示す(例: limbă /ˈlim.bə/)。
- 必要に応じて日本語話者向けの近似カナを補助的に付けてよいが、**カナだけで済ませない**(カナは近似であり正確な発音は IPA と解説で示す)。

## 4. 例文の標準フォーマット

すべてのルーマニア語例文には最低限、次を付ける。

| 項目 | 内容 |
|---|---|
| ルーマニア語原文 | DOOM3 準拠の正書法 |
| IPA/発音 | 音素表記 + アクセント |
| 自然な日本語訳 | 日本語として自然な訳 |
| 直訳 | 語順・構造が見える逐語的な訳 |
| 解説 | 文法・語彙のポイント |
| 使用場面 | いつ・誰に使うか |
| レジスター | 丁寧度(丁寧/中立/くだけた)・口語度・フォーマル度 |

表形式の場合の列構成:

```markdown
| ルーマニア語 | IPA/発音 | 自然な日本語訳 | 直訳 | 解説 | 使用場面・レジスター |
|---|---|---|---|---|---|
```

語彙表の列構成:

```markdown
| ルーマニア語 | 品詞 | IPA/発音 | 日本語訳 | 直訳/語源メモ | 例文 | 例文の日本語訳 | レベル | 使用場面 |
|---|---|---|---|---|---|---|---|---|
```

- 品詞略号: 名(名詞)、動(動詞)、形(形容詞)、副(副詞)、前(前置詞)、接(接続詞)、代(代名詞)、数(数詞)、間(間投詞)。名詞には性を付記: 名男 / 名女 / 名中。
- ルーマニア語の「中性」(neutru)は単数で男性・複数で女性として振る舞う点を、名詞関連の章では常に思い出させる。

## 5. 章の標準テンプレート

```markdown
# 第X章:タイトル【CEFR レベル】

## この章の目標
この章を終えると、次のことができるようになる。
- ...(3〜5項目、行動で書く)

## 前提知識
- ...(既習章への相対リンク)

## 本文解説
(日本語で段落として解説。見出しを分けてよい)

## 重要ポイント
- ...(3〜6項目)

## ルーマニア語例文
(§4 のフォーマット)

## 文法解説
(該当する場合)

## 語彙表
(§4 のフォーマット)

## よくある誤用
(日本語話者が特にやりがちな誤り。誤→正 の対で示す)

## ネイティブらしい言い換え
(教科書的表現 → 実際にネイティブが言う形)

## フォーマル/カジュアルの違い
(同じ内容の言い分け)

## 文化的注意点
(該当する場合)

## 練習問題
### 問題1 ...

## 解答
### 問題1 ...
(解答には「なぜそうなるか」の短い解説を付ける)

## 復習チェックリスト
- [ ] ...

## 次章への橋渡し
(次に学ぶ内容と、この章との接続を 2〜4 文で)
```

- A1〜A2 では練習問題と解答を同一ファイルに置く。B1 以降は分量に応じて `exercises/` `answer-key/` に分けてよい。
- すべての章がテンプレート全項目を持つ必要はない(会話章に「文法解説」がない等は可)が、**目標・例文・練習問題・解答・橋渡しは必須**。

## 6. 日本語の文体

- 解説文は「である/だ」体を基本とし、学習者への語りかけは「〜しよう」「〜してほしい」を許容する。
- 文法用語は初出時に「日本語名(ルーマニア語名または英語名)」を併記する。例: 接続法(conjunctiv)。
- 表記統一: 「ルーマニア語」(「ルーマニヤ」等は不可)、「モルドバ共和国」、数字は半角、章参照は相対リンク。

## 7. リンク・ファイル規則

- ファイル名: 小文字 + ハイフン + `.md`。レベル別本編は `NN-` の番号接頭辞。
- リンクはすべて**相対パス**。
- 各章の末尾に「前の章 / 次の章 / このレベルの目次」のナビゲーションリンクを置く。

## 8. 高リスク領域(医療・法律・行政・契約)

- 章冒頭に必ず免責ブロックを置く:

```markdown
> **免責**: 本章は語学学習を目的とした教材であり、医学的・法的助言ではない。
> 実務で使用する場合は、必ず資格を持つ専門家およびネイティブ話者の確認を経ること。
```

- 制度説明(医療制度・行政手続き等)は SOURCE_POLICY に従い出典を明記し、「〜年時点」の日付を付ける。
- 医療用語は誤訳リスクが高いものに ⚠ マークを付け、誤解例を明記する。

## 9. モルドバ共和国の扱い

- 言語名は「ルーマニア語」で統一(モルドバ共和国も 2023 年に憲法上の言語名をルーマニア語に統一)。
- 語彙・発音・社会言語的な差異は「モルドバでは〜」と明示して扱い、[culture/romania-and-moldova.md](./culture/romania-and-moldova.md) に集約する。
