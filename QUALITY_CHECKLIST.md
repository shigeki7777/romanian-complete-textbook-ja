# 品質チェックリスト

各章の執筆・レビュー時に使用する。すべての「必須」を満たさない章はマージしない。

## A. 教科書としての完成度(必須)

- [ ] 「この章の目標」が行動ベースで 3〜5 項目ある
- [ ] 前提知識と既習章へのリンクがある
- [ ] 本文が段落を持つ解説文である(箇条書きの羅列だけになっていない)
- [ ] 練習問題がある
- [ ] 解答があり、解答に短い解説が付いている
- [ ] 「次章への橋渡し」がある
- [ ] 章末ナビゲーション(前章 / 次章 / レベル目次)がある
- [ ] 章として独立して読める(この章だけ開いても文脈が分かる)

## B. ルーマニア語の品質(必須)

- [ ] 正書法が DOOM3 準拠(ș/ț はコンマ付き U+0219/U+021B、î/â の使い分け)
- [ ] すべての例文に IPA または発音注記がある
- [ ] すべての例文に自然な日本語訳と直訳の両方がある
- [ ] 例文にレジスター(丁寧度・口語度)の表示がある
- [ ] そのレベルで未習の文法・語彙を説明なしに使っていない
- [ ] 例文が実用的(実際に使う場面が想定できる)

## C. 日本語解説の品質(必須)

- [ ] 日本語話者の視点(日本語との対比・つまずきポイント)がある
- [ ] 文法用語の初出に原語併記がある
- [ ] 文体が STYLE_GUIDE §6 に従っている

## D. 深さ(推奨 — B1 以上は必須)

- [ ] よくある誤用(誤→正)がある
- [ ] ネイティブらしい言い換えがある
- [ ] フォーマル/カジュアルの対比がある
- [ ] 文化的注意点(該当する場合)がある

## E. 事実情報(該当する場合は必須)

- [ ] 歴史・制度・統計に出典がある(SOURCE_POLICY 準拠)
- [ ] 変化しうる情報に「〜年時点」がある
- [ ] 医療・法律・行政・契約の章に免責ブロックがある
- [ ] 誤訳リスクの高い医療・法律用語に ⚠ と誤解例がある

## F. 構造(必須)

- [ ] ファイル名が規約どおり(小文字ハイフン、本編は番号接頭辞)
- [ ] リンクがすべて相対パスで、リンク切れがない(`tools/check-links.py` で検査)
- [ ] cefr-map.md に登録されている
- [ ] 該当セクションの index.md からリンクされている
- [ ] 他章と重複・矛盾していない

## G. リリース前の横断チェック

- [ ] `python3 tools/check-links.py` がエラー 0
- [ ] `python3 tools/validate-markdown.py` がエラー 0
- [ ] PROGRESS.md が実態と一致
- [ ] CHANGELOG.md 更新済み
